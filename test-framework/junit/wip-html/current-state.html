ClassLoader Hierarchy in Quarkus Test Mode
JUnit 5 Lifecycle Callbacks in QuarkusTestExtension

1 LauncherSessionListener.launcherSessionOpened()

ConfigLauncherSession (registered via SPI in junit-config module):

  • Creates a SmallRyeConfigProviderResolver and registers initial test Config
  • Config registered against SystemClassLoader and TCCL
  • Sets LaunchMode.TEST during config build

CustomLauncherInterceptor (registered via SPI in junit module):

  • Creates a FacadeClassLoader(currentCl) with System CL as parent
  • Sets Thread.currentThread().setContextClassLoader(facadeLoader)
  • For Gradle: this happens before discovery starts, because Gradle loads classes early

2 LauncherDiscoveryListener.launcherDiscoveryStarted()

CustomLauncherInterceptor:

  • Ensures FacadeClassLoader is created and set as TCCL (idempotent)
  • Sets java.util.concurrent.ForkJoinPool.common.threadFactory to QuarkusForkJoinWorkerThreadFactory so ForkJoin threads also have the right TCCL

Discovery phase: JUnit scans for test classes using TCCL.loadClass(). Each call goes through FacadeClassLoader.loadClass() which:

  • Uses peekingClassLoader (a ParentLast URLClassLoader) to load a temporary copy for annotation inspection
  • If @QuarkusTest detected: creates/reuses a QuarkusClassLoader (Runtime CL) and loads the test class from it
  • If not a QuarkusTest: delegates to parent (System CL)
  • This is where CuratedApplication and StartupAction are created (augmentation happens here!)

3 LauncherDiscoveryListener.launcherDiscoveryFinished()

CustomLauncherInterceptor:

  • If not service-loader mechanism: resets TCCL back to original classloader
  • Validates that QuarkusTestProfileAwareClassOrderer is configured when multiple classloaders exist (critical for correct test ordering)
  • Throws if class orderer is misconfigured - tests with different profiles MUST be grouped together

4 ExecutionCondition.evaluateExecutionCondition()

AbstractJvmQuarkusTestExtension:

  • Checks quarkus.profile.test.tags config against QuarkusTestProfile.tags()
  • Disables tests whose profile tags don't match the configured tags

5 BeforeAllCallback.beforeAll(context)

QuarkusTestExtension:

  • Pushes test class onto currentTestClassStack
  • Sets LaunchMode.set(LaunchMode.TEST)
  • Calls ensureStarted(context) which calls doJavaStart(context) on first invocation:

doJavaStart() - the main bootstrap method:

  • Sets up hang detection timer (10 min default, configurable via quarkus.test.hang-detection-timeout)
  • Gets the QuarkusClassLoader from the test class (requiredTestClass.getClassLoader())
  • Gets the StartupAction from that classloader
  • Applies module configuration to augment and base runtime classloaders
  • Creates TestResourceManager (loaded from runtime CL for correct class visibility)
  • Starts test resources, overrides config with their properties
  • Populates callbacks and test method invokers (via ServiceLoader on runtime CL)
  • Calls startupAction.run() -> RunningQuarkusApplication
  • Stores ValueRegistry in JUnit extension store

6 InvocationInterceptor.interceptTestClassConstructor()

JUnit creates a test instance (from the class it discovered). The extension then:

  • Calls initTestState() -> createActualTestInstance()
  • Gets the CDI-managed instance: runningQuarkusApplication.instance(testClass)
  • Injects ValueRegistry values, HTTP resource URLs, and test resource manager fields
  • Stores as actualTestInstance (this is the real instance used for test method invocation)
  • For @Nested: creates outer instances and maintains outerInstances deque

7 BeforeEachCallback.beforeEach() + interceptTestMethod()

  • beforeEach(): pushes mock context, configures RestAssured URL, sets up test scope
  • interceptTestMethod(): calls runExtensionMethod()
  • runExtensionMethod() finds the equivalent method on the TCCL-loaded test class, then invokes it on actualTestInstance
  • JUnit's original invocation is skipped (invocation.skip())
  • Server-side exceptions are attached as suppressed exceptions on test failures

8 AfterEachCallback.afterEach() + AfterAllCallback.afterAll()

  • afterEach(): pops mock context, clears RestAssured state, tears down test scope
  • afterAll(): pops mock context, pops currentTestClassStack
  • Application stays running between test classes with the same profile
  • When profile changes: ensureStarted() detects it, closes old app (state.close()), calls doJavaStart() for new app

9 TestExecutionListener.testPlanExecutionFinished()

CustomLauncherInterceptor:

  • Resets TCCL to original classloader
  • Closes FacadeClassLoader (which closes peekingClassLoader, nulls runtime classloader refs)
  • Note: runtime classloaders are closed by the application shutdown, not by FacadeClassLoader
Test Discovery: How FacadeClassLoader Routes Classes

Important: Each class is essentially loaded twice during discovery:

  • peekingClassLoader (disposable URLClassLoader) - just for annotation inspection
  • QuarkusClassLoader (Runtime CL) - the actual class returned to JUnit

This double-loading has existed since Quarkus 1.2 when isolated classloaders were introduced. The peekingClassLoader uses a ParentLastURLClassLoader to avoid loading from parent first, ensuring annotations are inspected on the correct version of the class.

Application Bootstrap & Augmentation Sequence
CuratedApplication Creates ClassLoaders

getOrCreateAugmentClassLoader()

Name: "Augmentation Class Loader: TEST for ..."

Parent: System ClassLoader (baseClassLoader)

parentFirst: !isolateDeployment (false in test mode, so child-first)

Elements:

  • All dependencies from app model (except removed and reloadable)
  • Additional deployment archives
  • Banned: removed resources as MemoryClassPathElement
  • Parent-first: artifacts in configuredClassLoading.parentFirstArtifacts

createDeploymentClassLoader()

Name: "Deployment Class Loader: TEST for ..."

Parent: Augmentation ClassLoader

parentFirst: false (child-first)

Adds: application root paths, reloadable deps, additional archives

Purpose: gives build steps access to user application code

getOrCreateBaseRuntimeClassLoader()

Name: "Quarkus Base Runtime ClassLoader: TEST for ..."

Parent: System ClassLoader

parentFirst: false (child-first)

Elements:

  • Runtime-cp dependencies (non-reloadable)
  • Banned: app classes via ClassFilteredBannedElement (bans .class but not resources)
  • Resettable MemoryClassPathElement (for generated code)
  • Has CuratedApplication reference

Flat test classpath mode: if enabled, app root paths are added as normal elements (no restart needed)

createRuntimeClassLoader()

Name: "Quarkus Runtime ClassLoader: TEST for ... restart no: N"

Parent: Base Runtime ClassLoader

parentFirst: false (child-first)

Adds:

  • Generated resources (MemoryClassPathElement from augmentation)
  • Transformed classes (bytecode transformations)
  • Application root paths
  • Hot-reloadable additional archives
  • Reloadable workspace dependencies

This is the classloader test classes are loaded from.

Key Source Files

JUnit Integration (test-framework/junit/)

FacadeClassLoader.java

test-framework/junit/src/main/java/io/quarkus/test/junit/classloading/FacadeClassLoader.java

The critical classloader bridge. Set as TCCL before JUnit discovery. Intercepts loadClass() to route @QuarkusTest classes to QuarkusClassLoaders. Creates CuratedApplications and StartupActions during discovery.

CustomLauncherInterceptor.java

test-framework/junit/src/main/java/io/quarkus/test/junit/launcher/CustomLauncherInterceptor.java

Implements LauncherSessionListener, LauncherDiscoveryListener, TestExecutionListener. Creates FacadeClassLoader and manages TCCL throughout the JUnit platform lifecycle.

QuarkusTestExtension.java

test-framework/junit/src/main/java/io/quarkus/test/junit/QuarkusTestExtension.java

Main JUnit 5 extension. Implements all lifecycle callbacks. doJavaStart() bootstraps the app. Intercepts method invocations to route to CDI-managed test instances.

AbstractJvmQuarkusTestExtension.java

test-framework/junit/src/main/java/io/quarkus/test/junit/AbstractJvmQuarkusTestExtension.java

Base class for QuarkusTestExtension. Handles test profiles, execution conditions, Config initialization for non-system classloaders.

AppMakerHelper.java

test-framework/junit/src/main/java/io/quarkus/test/junit/AppMakerHelper.java

Creates CuratedApplications (resolves Maven/Gradle app model, builds QuarkusBootstrap). prepare() indexes test classes and creates augmentors. getStartupAction() runs augmentation.

Bootstrap & ClassLoading (independent-projects/bootstrap/)

QuarkusClassLoader.java

independent-projects/bootstrap/core/src/main/java/io/quarkus/bootstrap/classloading/QuarkusClassLoader.java

The custom classloader used in dev/test mode. Supports child-first loading, banned elements, parent-first elements, lesser-priority elements, resettable/transformed class memory elements. Linked to CuratedApplication.

CuratedApplication.java

independent-projects/bootstrap/core/src/main/java/io/quarkus/bootstrap/app/CuratedApplication.java

Factory for all classloaders: Augmentation CL, Base Runtime CL, Deployment CL, Runtime CL. Configures class partitioning based on dependency flags and ConfiguredClassLoading. Contains ClassFilteredBannedElement hack.

StartupAction.java

independent-projects/bootstrap/core/src/main/java/io/quarkus/bootstrap/app/StartupAction.java

Interface returned by augmentation. Provides the Runtime ClassLoader, run()/runMainClass() to start the app, config override, dev services initialization, module configuration.

RunningQuarkusApplication.java

independent-projects/bootstrap/core/src/main/java/io/quarkus/bootstrap/app/RunningQuarkusApplication.java

The running application handle. Provides getClassLoader(), instance() for CDI beans, getConfigValue(), valueRegistry(), close().

Config (test-framework/junit-config/)

ConfigLauncherSession.java

test-framework/junit-config/src/main/java/io/quarkus/test/config/ConfigLauncherSession.java

LauncherSessionListener that registers initial SmallRye Config before any test code runs. Registered against SystemClassLoader and TCCL.

Service Loader Registrations:

  • META-INF/services/org.junit.platform.launcher.LauncherSessionListener -> CustomLauncherInterceptor (junit module) + ConfigLauncherSession (junit-config module)
  • META-INF/services/org.junit.platform.launcher.LauncherDiscoveryListener -> CustomLauncherInterceptor
  • META-INF/services/org.junit.platform.launcher.TestExecutionListener -> CustomLauncherInterceptor
