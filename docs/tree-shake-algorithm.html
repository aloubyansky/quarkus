Quarkus JAR Tree-Shaking Algorithm

Build-time reachability analysis for removing unused dependency classes

1
Root Collection

Gather all classes that must be preserved regardless of bytecode references. These form the starting points for the reachability analysis.

Application
Main class, all app classes
Generated
Recorder bytecode, build-generated classes
Reflection & JNI
ReflectiveClassBuildItem, JniRuntimeAccessBuildItem
Service Providers
META-INF/services/ implementations
Native Image Config
reflect-config.json, jni-config.json, substitutions
String Constants
LDC strings in generated code matching dep class names
2
BFS Reachability Analysis

Breadth-first walk from roots through bytecode references. A single-pass ASM scan per class extracts all reference types simultaneously.

Type References
Superclasses, interfaces, field types, method signatures, exception types, inner/outer classes
Annotations & Generics
Annotation values (incl. class literals, arrays), generic type signatures
Instructions
NEW, CHECKCAST, field/method owners, LDC class constants, invokedynamic handles
Dynamic Loading
Class.forName(str), ClassLoader.loadClass(str) with string constants
ServiceLoader
ServiceLoader.load(Foo.class) → include all META-INF/services/Foo providers
JBoss Logging
_$logger, _$bundle, _impl companion classes
Sisu Named
Eclipse Sisu javax.inject.Named component classes
Serialization
Classes referenced in serialized resources (when ObjectInputStream + resource access detected)
Single-pass optimization: All the above extractions happen in one ClassReader.accept() call per class, replacing what was previously 4–5 separate ASM passes. Only classes with available bytecode are enqueued — JDK/platform classes are filtered out immediately.
3
Conditional Root Evaluation

Fixed-point loop over ReflectiveClassConditionBuildItem entries. When a condition type becomes reachable, its associated classes are added as new roots and traced via BFS. Repeats until no new classes are discovered.

4
Class-Loading Chain Analysis

Discovers classes loaded dynamically via ClassLoader.loadClass() or Class.forName() where class names are computed at runtime (e.g., via StringBuilder or invokedynamic string concatenation).

P1

Seed Propagation

Build a reverse caller index (callee → callers) over reachable classes in a single bytecode scan, incrementally extended on subsequent iterations. Only edges to non-JDK callees or class-loading seeds are stored (no forward callee map is needed). Starting from seed methods (ClassLoader.loadClass, Class.forName, MethodHandles.Lookup.findClass), propagate backwards via a worklist algorithm to find all application methods that transitively call a class-loading seed.

P2

Entry Point Discovery

Walk up the caller chain (depth-limited to 5 levels) to find classes whose <init> or <clinit> triggers the class-loading chain. Only dependency classes are considered — app and generated classes are excluded.

P3

Dynamic Execution in Forked JVM

New entry point classes are loaded and instantiated in a forked JVM process with an isolated RecordingClassLoader that captures every loadClass() attempt. If the object implements Map (e.g., BouncyCastle's Provider), its values are inspected for class name strings stored for deferred loading. Running in a separate process ensures thread-safety for parallel builds (-T2C) and full Metaspace reclamation on exit.

New classes
discovered?
← Yes

5
Post-Analysis Adjustments

After the main reachability analysis converges, apply adjustments for artifact exclusions and multi-release JARs before producing the final output.

Excluded Artifacts
Extensions can exclude entire JARs via JarTreeShakeExcludedArtifactBuildItem (e.g., BouncyCastle FIPS JARs that perform self-integrity checksums over their own classes).
Multi-Release Tracing
For reachable classes that have META-INF/versions/N/ bytecode targeting Java versions newer than the build JVM, extract class references from that bytecode and mark them reachable. Iterates to a fixed point. This ensures classes like BouncyCastle's ImportSSLSession_25 (referenced only by the version-25 SSLSessionUtil) are preserved.
6
Produce Final JAR

Compute per-dependency removal statistics. Exclude unreachable class files from the produced JAR (fast-jar, uber-jar, legacy-jar, or aot-jar). Non-class resources are always preserved.

Scope
Only dependency classes are candidates. App classes are never removed.
Multi-Release
Multi-release entries for reachable classes are preserved across all Java versions. Unreachable classes have all their versioned entries removed.
Granularity
Individual .class files. Resources, config files, and META-INF entries are preserved.
Reporting
One-line INFO summary with class count, dep count, and size saved. Debug logging for per-dependency breakdown.

Performance Optimizations

perf Single-Pass ASM Scan

All BFS reference types (class refs, string constants, ServiceLoader, sisu, serialization) extracted in one ClassReader.accept() per class instead of 4–5 separate passes.

perf Merged Bytecode Map

A single pre-merged map (app → dep → generated priority) replaces 3 sequential map lookups per class during BFS traversal.

perf BFS Reference Filtering

Only classes with available bytecode are enqueued. JDK/platform classes are filtered at the scan site, eliminating thousands of no-op queue iterations.

perf Caller-Index-Only Call Graph

Builds only a reverse caller index (callee → callers), not a forward callee map. Halves the call graph memory. Extended incrementally — subsequent iterations scan only newly discovered classes.

perf Worklist Seed Propagation

Replaces O(E × passes) fixed-point iteration with O(reachable-from-seeds) BFS worklist algorithm, with incremental updates across iterations.

perf Eager Memory Release

All analysis data structures (bytecode maps, call graph, service providers, etc.) are cleared immediately after stats computation via releaseAnalysisData(), freeing memory before the JAR packaging phase.

perf Skip Redundant Forks

Tracks executed entry points and classes discovered by prior forks. Skips JVM fork when new entry points were already captured.

safety Forked JVM Isolation

Phase 3 runs in a separate process, isolating System.out/err, security providers, system properties, and TCCL. Safe for parallel Maven builds (-T2C). Uses @argfile for classpath to avoid command-line length limits.

perf Object Reuse

BFS scan result object is cleared and reused across classes instead of allocating a new HashSet per class scan.
